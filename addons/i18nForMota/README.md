
# 适用于魔塔的翻译插件

词条有四种来源
1. 脚本:`tr('伤害')`
2. tscn:`xxx = "攻击"`
3. 表格:`该敌人周围{0}格范围内其他敌人攻击上升{1}%`
4. 事件:`\"这个是直接加属性的剑盾示例\"`

插件
+ 生成`1``2`的路径数组, 而后用godot自带的pot生成
+ 用脚本解决`3``4`的词条提取

## 本地化工具使用说明

直接双击cmd使用, 也可以在界面中点击按钮(默认不启用插件, 要先设置开启)

先 translations_pot_files.cmd 生成提取数组, 此时会提示要重新加载project.godot

点击 项目>项目设置>本地化>POT生成 点击生成POT. (覆盖)保存到locale/message.pot (比较慢)

此时完成了`1``2` (但是godot自带的pot生成不带行号.临时标成0,用来在poedit里跳转时不报错,有需求再识别行号)

再 extract.cmd 完成 `3``4`

日志在 addons/i18nForMota/i18n.log

配置在config.cmd中, 包含安装路径和po文件

### 安装以及配置

此程序调用了poedit, 需要安装3.5.1版本

https://download.poedit.net/Poedit-3.5.1-setup.exe

如果未安装在默认路径, 需要改 config.cmd 文件中的路径

> 其实只用了python 和 库polib. poedit会自带一个python3.12以及自带python库polib

## 后续

在.pot文件收集好了所有需要翻译的词条后，使用poedit打开.pot文件就可以打开进行查看
此文件包含了中文文本以及其原始位置, 在poedit中对着条目右键可以查看在源文件中的样子以及上下文

接着从模板文件创建各目标语言的.po文件,
编辑软件中会帮助检查文本和标点的格式, 以及给出一些翻译建议

随着游戏开发,模板文件重新生成后会改变增多条目, 此时可以打开对应目标语言的.po文件, 使用pot更新, 这样就可以把新增的词条合并进来, 同时所有词条的源位置也会更新
